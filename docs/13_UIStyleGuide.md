# 13 UI Style Guide

本文件记录《像素龙城》v0.4 起使用的像素 UI 风格规范。目标是让 HUD、背包、装备、NPC 对话和商店在视觉和交互上保持统一。

## 基准

- 设计分辨率：1600 x 900。
- HUD 固定在底部，基准高度约 145 px。
- 常用弹窗面板优先放在屏幕左右或下方，不遮挡玩家和主要交互目标。
- UI 第一版可以使用 `res://assets/ui/ui_atlas.png` 和 `res://assets/items/item_icons_sheet.png` 做原型占位，但正式切图需要后续精修。

## 材质与氛围

- 主材质：暗木、石、青铜、皮革。
- 面板应有经典 RPG 的厚重感，避免现代扁平、纯色、无边框的临时界面成为最终方向。
- 颜色整体保持低饱和，重要状态用清晰高对比色提示。
- 不直接复制任何商业游戏的 UI 图形、字体、按钮、布局细节或文本。

## HUD

- HUD 显示生命、魔法、经验、等级、金币和 6 个快捷栏槽位。
- HP 使用红色，MP 使用蓝色，EXP 使用金色或青绿色，金币使用金色。
- HUD 数值必须通过脚本接口更新，不直接依赖场景中的硬编码文本。
- 快捷栏 6 个槽位当前使用 64 x 64 px，以适配现有 atlas。

## HUD 响应式布局

- HUD 主布局统一使用 `Control` 坐标系，禁止使用固定位置的 `Sprite2D` 承担主布局。
- 底部 HUD 采用左、中、右三段结构：左区显示角色状态，中区显示快捷栏，右区显示等级、经验、金币和菜单入口。
- 现有 atlas 按左区 `(0..500)`、中区 `(500..1040)`、右区 `(1040..1420)` 裁成独立区域并分别缩放，避免整张底图在不同宽高比下整体拉伸。
- 左、中、右三区最小宽度分别约为 420、420、360 px；中区设置为可扩展区域，优先吸收窗口宽度变化。
- HP 和 MP 使用 `ProgressBar` 表达当前值与最大值，数值文本叠加在进度条上并保持可读。
- 快捷栏保持 6 个 64 x 64 px 槽位，在各目标窗口尺寸下不得挤压或错位。
- 右侧菜单入口使用 `flat` 图标按钮，在 atlas 图标上提供透明点击热点，并为每个按钮设置 tooltip。
- 响应式验收至少覆盖 1280 x 720、1600 x 900，以及嵌入窗口约 1498 x 842 或 1498 x 843。

## 背包与装备

- 背包第一版使用 5 列 x 6 行，共 30 格。
- 物品格基准尺寸：48 x 48 px。
- 图标优先使用 32 x 32 或 48 x 48；如果来自 sheet，先保证轮廓清楚，再做正式裁切。
- 装备槽位：武器、衣服、头盔、项链、戒指。
- 装备面板必须保留角色像素半身立绘区域，建议最小 220 x 260 px。

## 品质颜色

- 普通：白色或浅灰。
- 优秀：绿色。
- 稀有：蓝色。
- 史诗：紫色。
- 传说：橙色。

品质颜色用于物品名称、边框或小型标记，不应让整块面板变成单一强色。

## Tooltip

物品 tooltip 至少显示：

- 名称
- 类型
- 品质
- 描述
- 买入价格
- 出售价格

Tooltip 宽度第一版约 260 px，文本必须自动换行，不能遮挡当前正在操作的按钮。

## 按钮与状态

- 常用按钮包括关闭、下一句、买入、卖出、背包、装备。
- 命令含义不熟悉时按钮需要有可读文字；关闭、背包、装备等熟悉图标可以单独使用，但必须保留 tooltip 或可访问标签。
- 当前选中的物品格使用浅金或青铜色边框。
- 禁用状态降低亮度，但文字仍要可读。

## 面板规则

- 背包、装备、NPC 对话和商店属于主要模态面板。
- 同一时间默认只打开一个主要模态面板。
- 打开新主要面板时关闭其它主要面板。
- 每个主要面板必须有关闭按钮。
- UI Root 负责面板打开、关闭和互斥关系，避免面板之间直接互相调用。

## 数据规则

- NPC 数据来自 `data/npcs.json`。
- 商店数据来自 `data/shops.json`。
- v0.4 演示状态来自 `data/ui_demo_state.json`，不是正式存档。
- 正式物品仍以 `data/items.json` 为设计源。
- JSON id 使用稳定 snake_case，例如 `merchant_general_store`、`small_health_potion`。
