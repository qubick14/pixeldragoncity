# 02 Roadmap

## v0.1 文档与可移动原型

目标：创建 Godot 项目，玩家可以在测试地图中移动。

当前状态：已完成。实现记录见 `docs/DevLog.md`，原实施计划见 `docs/superpowers/plans/2026-06-28-pixeldragoncity-v0.1.md`。

验收标准：

- Godot 工程能打开。
- 主场景能运行。
- 玩家可用键盘移动。
- 相机跟随玩家。
- 有一张测试地图或测试地面。
- 确认斜俯视角色比例和相机缩放方向。
- 玩家占位图不再只是临时方块，应接近 48 到 64 像素高的地图角色比例。

## v0.2 基础战斗原型

目标：玩家可以攻击怪物，怪物可以追踪和受击死亡。

当前状态：已完成 headless 验证，仍需 Godot GUI 手动验收手感。实施计划见 `docs/superpowers/plans/2026-06-29-pixeldragoncity-v0.2.md`。

验收标准：

- 玩家攻击有范围判定。
- 怪物有生命值。
- 怪物会追踪玩家。
- 怪物死亡后消失。
- HUD 显示玩家生命。

## v0.3 数据驱动掉落与装备

目标：从 JSON 读取物品和怪物配置，支持掉落、背包和装备。

当前状态：已完成 headless 验证。实施计划见 `docs/superpowers/plans/2026-07-02-pixeldragoncity-v0.3.md`。v0.2 怪物死亡事件已经接入真实掉落生成。

验收标准：

- 读取 `data/items.json`。
- 读取 `data/monsters.json`。
- 怪物死亡后按配置掉落。
- 玩家可以拾取物品。
- 装备会改变玩家属性。
- 背包能区分可堆叠物品和装备。
- v0.3 逻辑有 headless 测试覆盖。

## v0.4 UI、NPC 与商店

当前状态：已完成 headless 验证，青木村村长、商人和铁匠已接入正式地图 NPC 交互；仍需 Godot GUI 手动验收布局、重叠、按钮和 tooltip 观感。实施计划见 `docs/superpowers/plans/2026-07-02-pixeldragoncity-v0.4-ui-npc-shop.md`。

目标：补齐基础 RPG UI 和村庄交互。

验收标准：

- HUD 显示生命、魔法、经验。
- 背包 UI 可打开和关闭。
- 装备 UI 可显示当前装备。
- NPC 对话可推进。
- 商店可以买卖基础物品。
- UI 采用原创像素面板，方向为暗木、石、青铜和皮革。
- 装备或角色面板预留像素半身立绘区域。

## v0.5 第一条完整新手流程

目标：完成青木村到黑狼林 Boss 的 10 分钟可玩内容。

验收标准：

- 玩家有明确出生点。
- 有新手任务。
- 有普通怪和 Boss。
- 有掉落和奖励。
- 可保存和读取。
- 从开始到完成任务不中断。
- 青木村和黑狼林至少有第一版原创地图美术或可替换 tileset。

## v1.0 方向

- 三职业雏形完整。
- 至少 5 张主要地图。
- 至少 30 件装备。
- 至少 20 种怪物。
- 至少 5 个 Boss。
- 至少 1 小时可玩流程。
- 三职业有原创地图 sprite 和像素半身立绘方向。
- HUD、背包、装备、商店和对话 UI 完成统一像素风格。
