# v0.4 UI 自适应整改设计

## 目标

修复 HUD、背包、装备、NPC 对话和商店界面的错位、遮挡与风格不统一问题。第一阶段优先修复 HUD，使其在不同窗口尺寸和 Godot 嵌入预览中保持稳定布局；后续面板沿用同一套响应式规则。

本轮仍限定在 v0.4 UI、NPC 对话和商店范围，不修改战斗、掉落、地图主线或职业系统。

## 当前问题与根因

### HUD

- `GeneratedUiFrame` 使用固定位置和缩放的 `Sprite2D`，按 1600 x 900 画布摆放。
- HP、MP、快捷栏、菜单按钮和状态文字使用带锚点的 `Control` 节点，会跟随实际窗口尺寸变化。
- Godot 嵌入预览窗口不一定是 1600 x 900，固定像素底图与响应式控件因此进入不同坐标系。
- HUD 底图已经包含视觉槽位，交互控件却没有按这些槽位建立统一分区，导致血条浮出底框、快捷栏偏移、右侧菜单和状态文字互相挤压。

### 其他面板

- 面板大量使用绝对偏移，缺少稳定的容器层级与统一内边距。
- 标题、内容、关闭按钮、tooltip 和半身立绘区域没有共享布局规则。
- 当前视觉依赖整张 UI atlas 拉伸，局部边框和内容区不能可靠适配面板尺寸。

## 方案选择

采用统一的 Godot `Control`、容器和锚点布局。

不再让固定坐标 `Sprite2D` 承担 HUD 主框架。HUD 底图改用底部全宽 `TextureRect` 或可拉伸面板资源，并与所有交互控件放在同一个响应式根节点下。

不使用脚本逐帧计算每个控件坐标。脚本只更新数值、状态和可见性，布局由场景中的容器与锚点负责。

## HUD 布局

### 根节点

- HUD 保持为 `CanvasLayer`，内部增加全屏 `Control` 根节点。
- 根节点使用全矩形锚点，所有 HUD 内容锚定到底部。
- HUD 视觉高度采用稳定区间，设计基准为 145 px。
- HUD 不参与游戏世界相机变换。

### 三段式结构

HUD 内容划分为左、中、右三个区域：

1. 左侧状态区
   - 基准宽度约 280 px。
   - 显示 HP、MP 和对应数值。
   - 血条、魔法条必须限制在各自轨道内部。
   - 数值文本不得改变轨道尺寸。

2. 中央快捷栏
   - 使用可伸缩区域承载 6 个固定尺寸槽位。
   - 快捷栏整体在可用中区居中。
   - 窗口变窄时先减小槽位间距，不缩放文字。
   - 槽位尺寸稳定，状态变化不能引发布局跳动。

3. 右侧信息与菜单区
   - 基准宽度约 420 px。
   - 等级、经验和金币放入独立信息行。
   - 背包与装备使用底图对应槽位内的可点击按钮。
   - 信息文字与按钮不得共享同一块自由定位区域。

### 响应式行为

- 1600 x 900：显示完整 HUD，保持设计稿比例和间距。
- 1280 x 720：保留三段结构，缩小中区间距，所有功能完整显示。
- Godot 嵌入预览：底框和交互控件使用同一可用宽度，不出现横向漂移。
- 宽屏窗口：左右区保持稳定宽度，中区吸收额外空间。
- 低于最小支持宽度时，优先压缩间距和次要留白，不允许控件重叠。

## 视觉资源处理

- 现有 `ui_atlas.png` 可以继续作为原型视觉来源，但不能再把整张固定区域当作不可伸缩 HUD。
- 第一阶段允许通过 `TextureRect` 的局部纹理或独立裁切资源完成布局。
- 若现有图块无法无损拉伸，应拆分左端、中段、右端资源，或改用 NinePatch 风格面板。
- 保持暗木、石、青铜和皮革方向，避免纯色临时矩形覆盖主要视觉区域。
- 所有像素资源保持 nearest 过滤，不进行模糊缩放。

## 其他面板整改规则

HUD 修复并通过验收后，按以下顺序处理：

1. 背包：标题栏、30 格网格、tooltip 和关闭入口。
2. 装备：五个装备槽、半身立绘预留区、属性区和 tooltip。
3. NPC 对话：头像区、姓名、正文、行为按钮和 HUD 避让。
4. 商店：买卖页签、商品列表、金币、tooltip 和关闭入口。

所有主要面板统一采用：

- 响应式根容器。
- 稳定标题栏与关闭按钮区。
- 明确内容区与内边距。
- 不嵌套无意义装饰卡片。
- tooltip 限制在可视区域内。
- 同一时间只显示一个主要模态面板。

## 数据与脚本边界

- `hud.gd` 继续负责生命、魔法、经验、等级、金币和快捷槽状态更新。
- UI 脚本不得根据 1600 x 900 硬编码绝对屏幕坐标。
- 数据接口和现有稳定 JSON id 保持不变。
- 本次整改不引入真实购买、出售、穿戴、战斗或任务逻辑。

## 验证方案

### 自动验证

- 保留并更新 `player_input_and_ui_test.gd` 与 `ui_panels_test.gd`。
- 增加 HUD 布局测试，检查关键节点在多种 viewport 尺寸下：
  - 位于可视区域内。
  - 不互相重叠。
  - 快捷栏保持 6 个槽位。
  - 左、中、右区域顺序稳定。

### GUI 验证

至少生成并检查以下尺寸截图：

- 1600 x 900。
- 1280 x 720。
- 当前 Godot 嵌入预览可用尺寸。

HUD 验收标准：

- HP、MP 位于底框左侧区域内。
- 快捷栏与底图槽位一致并保持居中。
- 等级、经验、金币可读且不遮挡按钮。
- 背包、装备按钮位于右侧菜单区。
- HUD 不遮挡玩家主要活动区域。
- 窗口尺寸变化后没有明显错位、裁切或重叠。

## 实施边界

第一轮只实施 HUD 自适应修复及其测试、截图和文档同步。HUD 验收通过后，再逐屏处理背包、装备、对话和商店，避免一次修改过多界面而难以定位回归。
