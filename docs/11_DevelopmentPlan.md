# 11 Development Plan

## 当前定位

《像素龙城》先做一个单机 2D 像素 ARPG MVP。核心体验是从村庄出发，到野外刷怪、获得掉落、装备成长、击败 Boss，再回到村庄交付任务。

画面方向采用原创像素美术，参考经典 2D 斜俯视 ARPG 的视觉语法：

- 斜俯视或近似 2.5D 的地图视角。
- 地图角色约 48 到 64 像素高，能看清职业、武器和怪物轮廓。
- UI 使用像素画面板，材质方向为暗木、石、青铜和皮革。
- HUD 以底部战斗信息栏为主，显示生命、魔法、经验、金币和快捷栏。
- 人物立绘先定为像素半身像用途，具体脸、服装和职业细节后续单独讨论。

禁止直接复制任何商业游戏素材、UI 图形、字体、地图块、怪物、角色造型、文本或音效。

## 版本计划

### v0.1 可移动原型

目标：项目能运行，玩家能在测试地图里移动，相机跟随。

开发内容：

- 创建 Godot 4 工程和主场景。
- 创建玩家场景、碰撞和移动脚本。
- 创建灰盒测试地图和边界。
- 确认斜俯视角色比例和相机缩放。
- 使用简单像素占位图验证角色大小。

验收标准：

- Godot 工程能打开并运行主场景。
- 玩家可用键盘八方向移动。
- 相机跟随玩家。
- 玩家不能走出测试地图边界。
- 角色占屏比例接近最终方向，不是临时小方块。

### v0.2 基础战斗原型

目标：玩家可以攻击野狼，野狼会追踪、受击和死亡。

计划文档：`docs/superpowers/plans/2026-06-29-pixeldragoncity-v0.2.md`

开发内容：

- 创建生命值组件。
- 创建近战攻击判定和受击判定。
- 创建野狼怪物场景和简单 AI。
- 添加受击反馈、血条和死亡流程。
- 定义玩家普通攻击的节奏和碰撞范围。

验收标准：

- 玩家攻击可以命中怪物。
- 怪物会追踪玩家并造成伤害。
- 玩家和怪物生命值会变化。
- 怪物死亡后消失或播放占位死亡效果。
- 战斗反馈足够清楚，能判断命中、受击和死亡。

### v0.3 掉落、背包与装备

目标：怪物死亡后产生掉落，玩家能拾取、查看和装备物品。

计划文档：`docs/superpowers/plans/2026-07-02-pixeldragoncity-v0.3.md`

开发内容：

- 读取 `data/items.json` 和 `data/monsters.json`。
- 校验怪物掉落 `item_id` 都存在于物品配置。
- 创建掉落物场景。
- 创建背包数据结构。
- 创建装备栏数据结构。
- 实现攻击、防御、生命等基础属性计算。
- 接入现有背包/装备菜单的最小数据显示。
- 准备第一批装备图标占位资源。

验收标准：

- 野狼死亡后按配置掉落金币、材料或装备。
- 玩家可以拾取掉落物。
- 背包能保存物品列表。
- 背包能区分可堆叠材料、消耗品和不可堆叠装备。
- 装备铁剑或皮甲会改变玩家属性。
- 数据修改后不用改代码即可调整掉落和属性。
- v0.3 数据、背包、装备、属性和掉落逻辑有 headless 测试覆盖。

### v0.4 像素 UI、NPC 与商店

目标：补齐基础 RPG UI 和村庄交互。

计划文档：`docs/superpowers/plans/2026-07-02-pixeldragoncity-v0.4-ui-npc-shop.md`

开发内容：

- 创建底部 HUD：生命、魔法、经验、金币、快捷栏。
- 创建背包 UI。
- 创建装备 UI，预留像素半身立绘区域。
- 创建 NPC 对话 UI。
- 创建商店 UI。
- 创建第一版 UI 像素风格规范。

范围边界：

- v0.4 可以使用演示 UI 状态验证界面，不阻塞于 v0.3 正式背包、装备和掉落数据。
- v0.4 不主动实现战斗、掉落、任务主线、职业技能、地图切换或保存读取，除非用户明确要求。
- 背包、装备、NPC 和商店数据都应通过稳定 JSON id 接入，不依赖聊天记录里的隐含设定。

验收标准：

- HUD 常驻显示战斗和成长信息。
- 背包和装备面板可打开、关闭和查看。
- NPC 对话可以推进。
- 商店显示买入、卖出、金币和基础物品价格，并预留交易信号。
- UI 风格统一，不使用现代扁平占位界面作为最终方向。
- v0.4 UI 层和青木村村长、商人、铁匠对话/商店入口已有 headless 测试覆盖，并已通过 Godot 正常渲染截图检查基础布局。

### v0.5 第一条新手流程

目标：完成青木村到黑狼林 Boss 的 10 分钟可玩闭环。

计划文档：`docs/superpowers/plans/2026-07-02-pixeldragoncity-v0.5.md`

开发内容：

- 制作青木村第一版地图。
- 制作黑狼林第一版地图。
- 创建新手 NPC 和第一个任务。
- 创建黑狼头目 Boss。
- 实现地图切换。
- 实现保存与读取。
- 用现有原创资源或灰盒占位验证关键路线。
- 接入最小掉落、背包和装备能力，使玩家能获得并装备铁剑或皮甲。

验收标准：

- 玩家从青木村出生。
- 玩家能从村长 `village_chief` 领取 `first_hunt`。
- 玩家能进入黑狼林，击败至少 3 只野狼和黑狼头目。
- 玩家能获得掉落并装备基础装备。
- 玩家能回村交付任务。
- 存档能记录角色、背包、装备、位置和任务进度，并已有 headless 写读验证。
- 从开始到完成任务不中断。

### v0.6 职业雏形

目标：从通用近战角色扩展到三职业雏形。

开发内容：

- 剑士：完善近战技能和防御节奏。
- 法师：实现火球术和魔法资源消耗。
- 灵术师：实现治疗或召唤雏形。
- 建立技能栏和冷却系统。
- 准备三职业地图 sprite 和半身立绘方向。

验收标准：

- 三个职业至少各有 2 个可用技能。
- 职业之间战斗距离和资源节奏有明显差异。
- 技能冷却和资源消耗有清楚反馈。

### v0.7 到 v1.0 内容扩展

目标：从 MVP 扩展到 1 小时可玩内容。

主要功能扩展：

- 更多地图：黑石矿洞、龙城王都、至少 5 张主要地图。
- 更多怪物：至少 20 种普通怪。
- 更多 Boss：至少 5 个 Boss。
- 更多装备：至少 30 件装备，包含品质颜色和基础词条。
- 更多任务：新手线之后加入村庄支线和矿洞线。
- 美术扩展：完整青木村、黑狼林、矿洞 tileset，三职业半身立绘，常用 NPC 头像。
- 音频扩展：攻击、受击、拾取、UI 点击和环境音。

## 主要系统拆分

### 角色与移动

- 玩家移动、碰撞、相机、动画状态。
- 怪物移动和 AI 状态机。
- 后续支持地图传送和出生点。

### 战斗

- 生命值、伤害公式、攻击判定、受击判定。
- 前摇、命中帧、后摇和冷却。
- 飘字、血条、死亡和掉落触发。

### 数据

- 物品、怪物、技能和地图使用 JSON 配置。
- Godot 运行时代码读取配置，不依赖聊天记录中的隐含设定。
- 配置 id 使用稳定字符串，例如 `iron_sword`、`wild_wolf`。

### 背包与装备

- 背包存储物品堆叠。
- 装备栏影响玩家属性。
- UI 显示物品名称、品质、属性和价格。

### UI 与 HUD

- HUD 常驻显示战斗和成长信息。
- 背包、装备、商店和对话使用独立面板。
- 人物立绘用于角色、装备和重要 NPC 对话，不阻塞早期功能开发。

### 地图与内容

- v0.1 使用灰盒测试地图。
- v0.5 前完成青木村和黑狼林第一版。
- 地图要有入口、出口、目标区域、怪物刷新点和边界。

### 美术生产

- 先用占位资源验证玩法。
- 再制作原创像素地图 tileset、角色 sprite、怪物 sprite、UI 面板和图标。
- 人物立绘作为独立美术任务推进，可反复修改。
- 资源状态按 `reference`、`prototype`、`blockout`、`production_candidate`、`production_ready` 管理。
- 每批资源需要在 `assets/ASSET_MANIFEST.md` 记录用途、状态、Godot 路径和下一步。
- 当前美术优先级是剑士动作观感、野狼 atlas、第一批物品图标、UI 拆件、黑狼林 tileset。

## 开发节奏

每个版本按以下顺序推进：

1. 更新或确认文档目标。
2. 实现最小可玩功能。
3. 用占位资源验证手感和比例。
4. 补充数据配置。
5. 替换关键美术资源。
6. 更新 `TODO.md` 和 `docs/DevLog.md`。
7. 运行 Godot 手动验收。
