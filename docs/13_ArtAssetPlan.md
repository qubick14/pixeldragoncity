# 13 Art Asset Plan

本文件是《像素龙城》像素美术资源的开发计划。目标是把现有方向稿和占位图推进成可切图、可导入、可测试、可长期扩展的 Godot 资源。

## 当前结论

项目已经有第一批原创像素方向稿和原型资源，详见 `docs/12_ArtDirection.md` 与 `assets/ASSET_MANIFEST.md`。这些资源已经能支撑原型展示，但还没有形成完整生产管线。

当前核心问题不是缺少所有图片，而是：

- 现有 AI sheet 不都符合严格网格。
- 部分资源是方向稿，不是最终游戏切图。
- 玩家、怪物、UI、图标和 tileset 的生产优先级还需要统一。
- 每批资源需要明确尺寸、帧数、命名、导入路径和验收标准。

## 资源状态分级

| 状态 | 含义 | 使用方式 |
| --- | --- | --- |
| `reference` | 方向参考图，不直接作为游戏切图 | 用于统一风格、角色气质和 UI 材质方向。 |
| `prototype` | 原型占位资源，可接入 Godot 验证比例和流程 | 可用于灰盒或早期版本，但后续需重切或精修。 |
| `blockout` | 动作或布局验证资源 | 用于验证方向、节奏、碰撞和 UI 结构，不代表最终美术精度。 |
| `production_candidate` | 规则网格或可复用切图候选 | 可进入实机验收，验收通过后再标记为正式资源。 |
| `production_ready` | 已通过尺寸、导入和实机验收 | 可作为当前阶段正式资源使用。 |

## 总体生产原则

- 所有正式资源必须原创，禁止复制商业游戏素材、UI、地图块、角色造型、字体、文本或音效。
- 先服务可玩闭环，再追求完整精修。
- 先定规格，再生成或重绘，避免反复猜裁切。
- 地图 sprite、半身立绘、怪物立绘可以分开制作，不要求由同一张图缩放。
- Godot 运行资源必须能通过 `ResourceLoader.load()` 或场景加载验证。
- 重要资源必须在 `assets/ASSET_MANIFEST.md` 记录状态、路径和用途。

## 文件与命名规则

源资源保留在根目录 `assets/`，Godot 可引用资源复制到 `godot/assets/`。

建议逐步采用：

```text
assets/
  references/
  sprites/
    swordsman/
    monsters/
    npcs/
  items/
  tilesets/
  ui/
  backgrounds/

godot/assets/
  sprites/
    swordsman/
    monsters/
    npcs/
  items/
  tilesets/
  ui/
  backgrounds/
  portraits/
```

命名规则：

- 使用 snake_case。
- 带版本号：`wild_wolf_walk_v1_atlas.png`。
- 带用途：`blockout`、`source`、`atlas`、`icon_sheet`。
- 不把方向稿命名为最终资源。

## 通用规格

### 地图角色

- 视觉高度目标：48 到 64 像素，当前剑士规格允许主体高度约 58 到 70 像素。
- 推荐正式 cell：`96x96`，脚底中心对齐。
- 方向：优先 8 方向移动，必要时保留 1 行 `idle_front`。
- 行走帧：每方向 4 帧起步。
- 攻击帧：每方向 4 到 6 帧起步。
- 受击帧：1 到 2 帧。
- 死亡帧：玩家可后置，怪物优先 4 到 6 帧。

### 怪物

- 野狼优先于其他怪物。
- 普通怪应能在地图上清楚区分轮廓、朝向、攻击范围和死亡状态。
- v0.5 阶段优先制作野狼、黑狼头目和掉落相关视觉。
- 怪物地图 sprite 与怪物立绘分开规划。

### 物品图标

- MVP 图标建议统一 `32x32`，保留更高清源图也可以。
- 图标 sheet 必须严格网格。
- 品质颜色遵循 `common` 白、`uncommon` 绿、`rare` 蓝、`epic` 紫、`legendary` 橙、`mythic` 红。
- 图标优先保证轮廓可读，不追求过多细节。

### UI

- UI 采用暗木、石、青铜、皮革材质方向。
- 面板资源优先支持 9-slice 或可复用切片。
- HUD、背包、装备、对话、商店应共用同一套边框和格子语言。
- 文字区域必须留白，不能让纹理影响可读性。

### Tileset

- 推荐以 `32x32` 为第一版生产基准。
- 每个 tileset 需要区分可行走、不可通行、装饰、交互点和地图出口。
- 黑狼林和青木村优先于黑石矿洞。
- tileset 允许不透明背景，但需要清楚记录使用方式。

## 阶段计划

### Art-0 资源审计与规格同步

目标：把现有资源按状态分类，并补齐文档入口。

产出：

- `docs/13_ArtAssetPlan.md`
- `docs/superpowers/plans/2026-07-02-pixeldragoncity-art-assets.md`
- `assets/ASSET_MANIFEST.md` 状态说明
- `TODO.md` 美术专项任务
- `prompts/art.md` 标准输出格式

验收：

- 后续 AI 能直接从文档理解美术开发顺序。
- 每类资源都有状态、目标规格和下一步。

### Art-1 玩家与野狼战斗资源

目标：支撑战斗和新手流程观感。

优先资源：

- 剑士正式行走候选 atlas。
- 剑士普通攻击 atlas。
- 剑士受击帧。
- 野狼 idle、walk/run、attack、hurt、death。
- 黑狼头目 blockout 或临时 sprite。
- 血条、伤害数字和命中反馈小资源。

验收：

- 玩家移动不再像单张贴图平移。
- 普通攻击方向、命中时机和受击反馈可读。
- 野狼追踪、攻击、死亡能在实机中一眼识别。

### Art-2 掉落与装备图标

目标：支撑掉落、背包和装备闭环。

优先资源：

- 木剑、铁剑。
- 布衣、皮甲。
- 草药、小型生命药水。
- 金币、狼皮、狼牙、黑色晶片。
- 地面掉落光点或小图标。

验收：

- 背包网格中能区分装备、消耗品、材料和金币。
- 地面掉落物能被玩家发现。
- 品质颜色和物品类型在小尺寸下仍可读。

### Art-3 UI Atlas 与面板拆件

目标：支撑 v0.4 HUD、背包、装备、对话和商店。

优先资源：

- HUD 底板。
- 生命、魔法、经验条。
- 快捷栏格子。
- 背包格子、装备槽、选中框。
- 对话框、商店面板、按钮、关闭按钮。
- 金币小图标。

验收：

- UI 风格统一，不是现代扁平占位界面。
- 面板可复用，不需要每个窗口重画一套。
- 字体和数字区域清晰。

### Art-4 青木村与黑狼林 Tileset

目标：支撑 v0.5 第一条新手流程。

优先资源：

- 青木村地面、木屋、围栏、路牌、商人摊位、铁匠铺。
- 黑狼林草地、泥路、树、灌木、石头、草药点、Boss 区域边界。
- 村口、森林入口、矿洞入口。

验收：

- 玩家能清楚看出入口、出口、目标区域和边界。
- 地图不会让玩家迷路。
- 路径和障碍优先于装饰密度。

### Art-5 职业、NPC 与半身立绘

目标：支撑 v0.6 职业雏形和重要 NPC 对话。

优先资源：

- 剑士、法师、灵术师半身像。
- 村长、商人、铁匠、新手任务 NPC。
- 黑狼头目和矿洞 Boss 方向立绘。

验收：

- 三职业有明显轮廓、配色和武器/法器识别。
- NPC 能在对话 UI 中承担身份识别。
- 半身像不阻塞地图 sprite 和系统实现。

## 当前最高优先级

1. 确认剑士当前 blockout atlas 的方向和步行动作是否通过 GUI 观感验收。
2. 产出野狼正式或半正式 atlas，替换临时怪物视觉。
3. 产出第一批 `32x32` 物品图标规则 sheet，服务背包和掉落。
4. 拆分 UI atlas 的 HUD、格子、按钮和面板资源，服务 v0.4。
5. 制作黑狼林第一版 `32x32` tileset，服务 v0.5 路线和战斗区域。

## 文档维护规则

- 美术方向变化：更新 `docs/12_ArtDirection.md`。
- 美术资源计划变化：更新本文件。
- 具体资源状态变化：更新 `assets/ASSET_MANIFEST.md`。
- 新资源生成提示词变化：更新 `prompts/art.md`。
- 任务完成或推迟：更新 `TODO.md` 和 `docs/DevLog.md`。
- 单个资源规格较复杂时，写入 `docs/art/` 下的独立规格文档。
